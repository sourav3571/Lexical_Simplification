========================================================================
                     DATAFLOW AND COMPONENT ROUTING
========================================================================

The simplified input sentence goes through the following file-to-file path:

1. ENTRY POINT:
   - `interactive_simplifier.py` accepts input from the terminal or CLI prompt.
   - Passes the string to `ai_simplifier.py` (which acts as the main pipeline orchestrator).

2. STAGE 1: PREPROCESSING
   - `ai_simplifier.py` -> `preprocessing.py`
   - Runs the raw sentence through spaCy to lemmatize tokens, extract POS tags, and isolate content words (nouns, verbs, adjectives, adverbs, proper nouns).
   - Returns: A list of `content_tokens` containing character offsets and POS tags back to `ai_simplifier.py`.

3. STAGE 2: COMPLEX WORD IDENTIFICATION (CWI)
   - `ai_simplifier.py` -> `dynamic_cwi.py`
   - Uses the fine-tuned DeBERTa CWI Classifier (`cwi_model.py`) loading checkpoint weights from `cwi_bert_figurative_v2.pt`.
   - Utilizes hyperparameters defined in `cwi_config.py`.
   - Invokes `bert_complexity.py` to extract features:
     - Inherent MLM Bias (30%)
     - Semantic Embedding Drift (50%): Measures deviation from concrete templates via `embedding_store.py` (which caches vector states).
     - MLM Prediction Spread (20%)
   - Applies dynamic standard deviation thresholds and POS-aware rarity overrides (bypassing simple thresholds for low-frequency nouns/adjectives/verbs).
   - Returns: Identified complex target words to `ai_simplifier.py`.

4. STAGE 3: WORD SENSE DISAMBIGUATION (WSD)
   - `ai_simplifier.py` -> `bert_sense_disambiguator.py`
   - Performs cosine similarity checks between the contextual target embedding and WordNet gloss embeddings.
   - Returns: The target synset corresponding to the correct sense to `ai_simplifier.py`.

5. STAGE 4: CANDIDATE GENERATION & INITIAL FILTERING
   - `ai_simplifier.py` -> `bert_candidate_generator.py`
   - Queries WordNet relations and BERT MLM predictions.
   - Performs structural sanity filtering (POS matches, Zipf frequencies via the `wordfreq` package).
   - Returns: Filtered list of simple substitution candidates to `ai_simplifier.py`.

6. STAGE 5: CANDIDATE RANKING
   - `ai_simplifier.py` -> `bert_ranker.py`
   - Employs a Gated Fusion Neural Ranker (PyTorch model) using serialized weights in `gated_fusion_ranker.pt`.
   - Scores candidates based on MLM probability, word embedding cosine similarity, surprisal reduction (`bert_surprisal.py`), and sentence fluency changes.
   - Returns: An ordered list of ranked candidates to `ai_simplifier.py`.

7. STAGE 6: 4-GATE VALIDATION
   - `ai_simplifier.py` -> `bert_validator.py`
   - Sequentially validates top candidates against Gate 1 (probability), Gate 2 (word/sentence similarity), Gate 3 (sentence fluency log-likelihood), and Gate 4 (CWI score reduction).
   - Returns: The single validated word substitution (or None) to `ai_simplifier.py`.

8. STAGE 7: PARALLEL SUBSTITUTION
   - `ai_simplifier.py` -> `parallel_replacer.py`
   - Performs concurrent replacements across multiple non-overlapping word offsets and handles title/capitalization casing.
   - Returns: The final simplified sentence back to `interactive_simplifier.py`.


========================================================================
                          DATASETS USED
========================================================================

The following corpora were used to calibrate, train, and validate the modules:

1. CWI 2018 Shared Task Dataset
   - Primary corpus (~27,000 tokens) with annotations representing binary and probabilistic word complexity. Used for baseline training of the DeBERTa model.

2. LexMTurk Dataset
   - A dataset of 500 English sentences annotated by crowdsourced human workers. Used to extract gold substitution pairs for training and validation.

3. BenchLS Dataset
   - A standard lexical simplification benchmark of 929 sentences. Used for training features and pipeline-wide verification.

4. Augmented Figurative/Literal CWI Dataset
   - Custom dataset (`augmented_cwi_data.json`) consisting of 200 samples (100 figurative, 100 literal) using highly polysemous words (e.g., "nature", "spirit", "face", "heart").
   - Configured with `generate_augmented_data.py`.
   - Specifically used in `retrain_cwi.py` to teach the DeBERTa model to recognize abstract/figurative senses as complex (e.g., "nature" of a person) while ignoring literal senses (e.g., "nature" outside).
